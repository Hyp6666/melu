让 Claude Code 拥有
跨会话记忆

Melu 是一个轻量级的透明代理，旨在解决 Claude Code 每次重启都会“失忆”的问题。它能够自动提取关键上下文并实现语义化检索。

查看 GitHub

# Install the global package

npm install -g @hope666/melu

# Initialize Melu environment

melu init

# Run Claude Code with memory

melu run -- claude

✓ Memory bridge established

✓ Claude Code process started

Claude Code 很强，但它默认没有跨会话、跨账号、跨设备的长期记忆能力

history

会话碎片化

每次重启 Claude Code，它都会忘记之前的讨论。你需要重复输入背景信息，浪费大量 Token 和时间。

psychology_alt

核心偏好丢失

个人信息与偏好等信息细节无法跨会话持久化保留，导致回复不一致，各账号各API无法有效链接个人记忆

cloud_off

私有化顾虑

标准的云端记忆往往不可控，用户需要一个本地化、透明且安全的记忆中间层。

Melu 做了什么

router

1. Proxy

拦截 Claude Code 与 API 的通信，具有注入历史记忆感知的能力。

memory

2. Injection

根据当前对话意图，动态从本地向量库检索并注入历史上下文。

class

3. Extraction

用户发送出Prompt后，自动提取结构化知识并更新长期记忆库。

核心能力

verified 专业级工具集

怎么用

melu init

# 初始化配置

melu run -- claude

# 默认启动

快速上手

只需两行命令。`init` 会帮你设置好本地的环境变量和代理，`run` 则会自动启动代理服务并拉起 Claude Code 进程。

多记忆库管理

你可以为不同的项目指定不同的记忆库。通过隔离存储，避免不同项目间的代码逻辑发生干扰。

melu list

# 查看所有记忆会话

melu run --db custom-proj

# 切换项目空间

shield_lock

为什么它是可信的

Melu 的设计原则是“透明”。作为一个中间层，它并不存储你的 API Key（它仅通过环境变量传递），也不将你的原始代码上传到除了 Anthropic 以外的任何地方。所有的记忆索引都存储在本地 SQLite 和本地 Embedding 模型中。

工作方式

Embedder Daemon

运行于本地的极小模型服务，负责将对话片段转换为向量坐标。

Proxy

基于 Node.js 的高性能 HTTP 拦截器，负责上下文的静默注入。

Extractor Worker

定期对历史记录进行归档和去噪，确保记忆库的精简与高效。

list_alt 运行要求

  • Node.js 20+

    推荐使用最新的 LTS 版本以获得最佳性能。

  • Claude Code

    需已在全局安装 `@anthropic-ai/claude-code`。

  • 600MB 磁盘空间

    主要用于存储本地向量数据库及其索引文件。

常见问题

Melu 会导致 Claude Code 的回复变慢吗？ expand_more
会有非常轻微的额外延迟，通常只有几十毫秒，主要来自本地记忆检索。但在正常使用中，这种延迟基本不会影响你的体验。
它可以和现有的 `.claude` 配置共存吗？ expand_more
可以。Melu 作为代理层工作，不会干扰 Claude Code 读取自己在 `.claude` 目录下的配置。
如果我更换了电脑，记忆可以同步吗？ expand_more
目前记忆文件存储在本地的 `~/.melu` 目录下。你可以手动备份该目录，或者将其放置在 iCloud/Dropbox 等同步盘中进行跨设备共享。
